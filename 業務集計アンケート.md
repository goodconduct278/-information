# 業務知識・引継ぎ状況に関するアンケート（統合版）

## 1. 目的

本アンケートは、部内の業務知識・引継ぎ状況を整理し、今後の業務継続性向上、教育、引継ぎ負担軽減に役立てることを目的とする。

特定の個人や担当業務を問題視するものではなく、退職・異動・担当変更・長期不在時にも業務を安定して継続できるよう、現在どのような知識が必要とされているかを確認するためのものである。

回答内容は、今後の業務知識DB化、手順書整備、引継ぎ資料作成、教育資料整備の検討材料として使用する。

## 4. 統合版アンケートの考え方

これまでの簡易版・詳細版を分けず、1つのアンケートに統合する。

### 基本方針

- 前半は全員が回答しやすい基本項目にする
- 後半は、該当する業務がある場合のみ記入する深掘り項目にする
- 同じ内容を何度も聞かない
- 集計しやすいように、業務ごとに横並びで回答できる形式にする
- アンケート後に、優先度の高い業務だけ追加ヒアリングする

### 想定回答時間

- 通常回答：10〜15分程度
- 深掘り項目まで記入する場合：20分程度

# 18. 集計時に見るべきポイント

アンケート回収後は、回答をそのまま読むだけではなく、次の観点で整理する。

| 見る項目 | 意味 |
|---|---|
| 代替担当者がいない業務 | 属人化リスクが高い |
| 資料がない業務 | 知識喪失リスクが高い |
| 資料はあるが古い業務 | 誤対応リスクがある |
| 判断基準が不明な業務 | 引継ぎ困難 |
| 過去トラブルがある業務 | 優先的にDB化すべき |
| 客先・取引先ごとの注意点が多い業務 | 暗黙知が多い |
| 口頭確認が多い業務 | 形式知化が必要 |
| 個人フォルダやメールに資料がある業務 | 退職・異動時に消失しやすい |
| 過去経緯が必要な業務 | 後任者が判断を誤りやすい |

# 19. 優先度判定の簡易スコア

各業務について、以下の項目に該当するかを確認し、点数化する。

| 項目 | 点数 |
|---|---:|
| 代替担当者がいない | 3点 |
| 資料・マニュアルがない | 3点 |
| 判断基準が複雑 | 3点 |
| 過去トラブルがある | 2点 |
| 顧客・取引先ごとの注意点が多い | 2点 |
| 発生頻度が高い | 2点 |
| 業務影響が大きい | 3点 |
| 資料が個人フォルダやメールにある | 2点 |
| 資料が古い・最新版が分かりにくい | 2点 |
| 過去経緯を知らないと対応が難しい | 3点 |

## 判定目安

| 合計点 | 優先度 |
|---:|---|
| 10点以上 | 最優先 |
| 7〜9点 | 優先 |
| 4〜6点 | 必要に応じて対応 |
| 3点以下 | 後回し |

# 20. アンケート実施時の注意点

## 20-1. 言い方に注意する

避けた方がよい表現：

- 属人化している人を調査します
- 誰が仕事を抱え込んでいるか確認します
- 引継ぎができていない業務を洗い出します

推奨する表現：

- 業務知識を整理します
- 引継ぎ負担を軽減します
- 業務継続性を高めます
- 将来の退職・異動に備えます
- 後任者や若手が困らないようにします

---

## 20-2. 個人批判に見えないようにする

本アンケートは、特定の人を問題視するものではない。  
個人名を出す場合も、「その人しかできないこと」を責めるのではなく、「会社として残すべき重要知識」として扱う。

---

## 20-3. 最初から完璧を目指さない

初回アンケートの目的は、完全な業務一覧を作ることではない。  
まずは、以下を把握できれば十分である。

- どの業務が危なそうか
- 誰に詳しく聞くべきか
- どの資料が不足しているか
- どの業務を試作対象にすべきか

---

# 21. 実施後の流れ

アンケート実施後は、次の流れで進める。

1. 回答を集計する
2. 業務ごとに分類する
3. 属人化リスク・知識喪失リスクを点数化する
4. 優先度の高い業務を3〜5件選ぶ
5. 該当業務の担当者に追加ヒアリングを行う
6. 業務知識DBのサンプルページを作成する
7. 実際に別担当者が使えるか確認する
8. 上司に試作結果を報告する
9. 部内展開または正式導入を検討する

---

# 22. 追加ヒアリング時に聞く内容

アンケートで優先度が高いと判断された業務について、担当者に追加ヒアリングを行う。

## 聞き取り項目

- この業務で新人が一番ミスしやすいところはどこか
- 過去に大きく揉めた案件や手戻りはあるか
- この業務で絶対に確認すべき相手は誰か
- 資料には書いていないが、実際には気をつけていることは何か
- この判断を間違えると何が起きるか
- 退職・異動した場合、一番困る部分はどこか
- 昔と今でやり方が変わった点はあるか
- なぜ今のやり方になったのか
- この客先・取引先で注意すべきことはあるか
- 過去資料を見るなら、どのフォルダ・どの案件を見るべきか

# 23. 上司への説明文例

> 業務知識DBを作成するにあたり、私自身がすべての業務内容を正確に把握しているわけではないため、まずは部内の業務知識・引継ぎ状況をアンケート形式で洗い出したいと考えています。  
> 目的は、個人の業務を問題視することではなく、退職・異動・担当変更時にも業務を継続できるよう、どの業務にどのような知識が必要かを把握することです。  
> 以前は簡易版と詳細版に分けていましたが、内容の重複が多いため、今回は1つの統合版アンケートにまとめ、全員が回答する基本項目と、該当業務のみ記入する深掘り項目に分けています。  
> 回答結果をもとに、優先度の高い業務を3〜5件選び、追加ヒアリングと業務知識DBの試作につなげたいと考えています。

# 24. 一言でのまとめ

この統合版アンケートは、簡易版・詳細版を分けずに、以下を一度で確認するためのものである。

- どの業務があるか
- 代替できる人がいるか
- 資料やマニュアルがあるか
- 資料だけで対応できるか
- 判断基準や過去経緯が属人化していないか
- トラブルや手戻りが起きやすい業務は何か
- 優先的に業務知識DB化すべき業務は何か

最初の目的は、完璧な業務一覧を作ることではない。  
まずは、退職・異動・長期不在時に困る可能性が高い業務を見つけ、重要業務3〜5件の試作につなげることである。
